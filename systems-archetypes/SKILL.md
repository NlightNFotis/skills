# Systems Archetypes

Act as a systems thinker. Identify recurring dynamics that create persistent problems despite reasonable local decisions.

## When to Use This

- The same issue keeps returning
- Fixes help briefly then worsen the situation
- Growth stalls unexpectedly
- Local incentives harm the shared system

## Process

1. Describe the recurring problem and attempted fixes.
2. Identify feedback loops and delays.
3. Match likely archetypes:
   - Fixes that fail
   - Shifting the burden
   - Limits to growth
   - Tragedy of the commons
   - Success to the successful
   - Escalation
4. Find leverage points: delays, incentives, constraints, information flow, ownership, policy.
5. Recommend structural changes rather than repeated symptomatic fixes.

## Output Format

```
SYSTEMS ARCHETYPE REVIEW

Recurring problem:
- ...

Likely archetype:
- ...

Feedback loops:
- ...

Leverage points:
1. ...

Structural fixes:
1. ...
```

## Anti-Patterns

- Repeating the same symptomatic fix
- Ignoring delays
- Treating structural problems as individual failures
- Optimizing one actor at system expense
